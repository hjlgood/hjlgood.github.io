Why I Chose Hexo

November 28, 2021 blog Page viewed Times

In this blogpost, I would like to share my experience of building a blog using Hexo.

What is Hexo

Hexo is a static-site blog framework based on Node.js. You can use MARKDOWN to write your blog posts and deploy it on Github-Pages or GitLab-Pages using Hexo.

Why Hexo

There are plenty of other blogging tools and websites, most notably Wordpress, but I chose Hexo to build my website. Here are a couple of reasons why I chose Hexo over other platforms.

1. You own your own content

A long time ago when I was in college, I made a blog using Blogger to save some of my work online. It included some texts and images of my assignments so I could download it in case I forgot to bring a USB stick. Few years later after graduation, I found out that Google removed all of my images from the blog due to ‘inactivation’. Not only was I disappointed, but also there was nothing I could do to retrieve those contents. This is because when you upload your content online, you’re relying on the service provider to keep you contents safe. If you use Hexo, you can keep all your source in your local machine where you can keep it safe.

2. GitHub provides free hosting

Hexo provides quite a few options when deploying your website. Perhaps the most common one is GitHub, but there are also other options such as Heroku, Netlify, Rsync, OpenShift, FTPSync, SFTP, Vercel, 21YunBox, Bip, or RSS3. If you have your own hosting server, you can copy your contents from the “public” folder and paste it in your server. Personally, I chose GitHub because GitHub is by far the most common website where poeple store their code. (It’s also free!)
As logn as you pay for the domain name for your website, you’re good to go.

3. Markdown support

Hexo supports Markdown file by default. you can use your favorite editor to write your blog posts and save it as .md file in your local machine. I personally use Visual Studio Code alongside with Foambubble to use this place as my personal Zettelkasten.

4. Foambubble

Foam is an extension to Visual Studio Code editor, arguably one the most famous text editors for programmers. It provides some interesting features like markdown template and wikilinks which helps you navigate through your files. Explaining what Foam is requires a whole separate post, but in essence, it’s a tool that generates relationships between your ideas. Here is an example of a well developed graph created using Foam.

5. Plethora of themes

Hexo supports plenty of themes and templates using .ejs or .swig. You might want to check out some free themes created by Hexo community at here.

Conclusion

This was a short post to introduce to you some of the reasons why I decided to use Hexo as my blogging framework. In the following post, I’ll explain how to set up a blog using Hexo and point out some of the tricky parts that you may encounter. Hope you enjoy.
